# pacman_duel

## 简介

这是一个可对战的pacman游戏。一方控制吃豆人，另一方控制一个史莱姆。吃豆人的任务是吃掉所有的豆子，史莱姆的目的是追上吃豆人。史莱姆有一个小助手，这个小助手在每一刻都会计算跟吃豆人的路径并选择最短路径（BSF）。

玩家可以控制任意一方，另一方则由AI控制。玩家也可以当观众，把吃豆人和史莱姆都交给AI操纵。

这个项目的主要目的是，帮助玩家理解reinforcement learning。

## 用户需求

 - 用户可以选择当吃豆人或当史莱姆，让AI选另一方。也可以让AI互怼。
 - AI操作的那一方，还有附加选项，选择算法。
 - 提供进阶版模式，在这个模式下，每个算法可以调节参数。参数菜单取决于是什么算法。

## 前端

 - 可以在网页运行，也可以是个本地图形界面。如果本地，必须支持wayland。
 - 在进阶版模式下，每种控制方式一旦选定，就会显示胜率。
 - 玩家用上下左右控制吃豆人
 - q或者esc退出游戏，回到主菜单

## 游戏

 - 游戏的部分是标准pacman的棋盘，但是只有两个敌人，而不是三个。
 - 只有小豆子。没有大豆子，所以吃豆人没有反攻的选项。
 - 吃豆人胜利条件是吃完豆子。
 - 敌人胜利条件是，史莱姆或者它的助手追上吃豆人。
 - 如果史莱姆或者助手追上吃豆人的瞬间，吃豆人刚好把豆子吃完，判定为吃豆人赢。

## 后端

 - AI有不同算法
   - 随机：到处乱跑
   - 最短距离：BSF,和助手一样
   - 跟屁虫：开局先到达吃豆人的初始位置，并开始完全重复吃豆人走过的路径。包括停顿也一样复制。
   - Reinforcement learning
     - 手搓版本
     - 市面上现成的package版本
 - 每次游戏的结果保存。统计胜率用以往数据，而不是通过模拟。

## 可延展性

目前不打算支持，但今后有可能支持的功能。设计当前版本的时候应该把向前的路留着，方便添加新功能。

 - 跨平台
 - 网页版本，部署到类似于AWS的服务。
 - 允许用户添加自己的ML模型
 - 允许用户提供不同的棋盘
 - 可添加史莱姆的数量
